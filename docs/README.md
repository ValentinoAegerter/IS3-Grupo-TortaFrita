En este directorio se encuentran los ADRs (Architecture Decision Records) para el proyecto.

Archivos incluidos:
- ADR-001-Postgres-ValentinoAegerter.md: ADR que documenta la elección de PostgreSQL como motor de base de datos.
- ADR-002-Cache-Scaling-NicolasFirka.md: ADR propuesto que documenta la introducción de Redis para caching y control de concurrencia.

Issues sugeridos
- Nombre del issue para Valentino Aegerter: "ADR: Selección de motor de BD (PostgreSQL) - ValentinoAegerter"
- Nombre del issue para Nicolas Firka: "ADR: Introducción de Redis para caching/escala - NicolasFirka"

Flujo recomendado
1) Crear un issue por integrante usando los nombres sugeridos arriba. En el cuerpo del issue incluir un enlace al archivo correspondiente en /docs (ej. docs/ADR-001-Postgres-ValentinoAegerter.md).
2) Asignar el issue al integrante correspondiente y marcarlo como "Open". Cuando el ADR esté listo y commiteado en el repo, cerrar el issue.
